## Service Communication Patterns

| Pattern | Description |
|---------|-------------|
| **API Gateway вҶ’ Microservices** | HTTP proxy via restpoint-gateway |
| **Microservices вҶ’ Database** | Direct MySQL/MariaDB connections via mysql2 |
| **Microservices вҶ’ Cache** | Redis connections via ioredis |
| **Real-time** | SocketIO for live updates |
| **Notifications** | BullMQ queues for async notification processing |
| **Service Discovery** | Environment variable based (docker compose service names) |

## Network Configuration

- **Docker Network**: `restpoint_net` (bridge driver)
- **Internal DNS**: Docker Compose service name resolution
- **Exposed Ports**: All services exposed via gateway on port 5000
- **Frontend**: Nginx serves SPA, proxies API to gateway

## Health Check Endpoints

- All services expose `GET /health` returning `{"status":"ok","service":"name","version":"1.0.0"}`
- Gateway exposes `GET /api/v1/health` with aggregated status
- Docker health checks configured on critical services
